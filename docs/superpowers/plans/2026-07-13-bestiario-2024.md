# Plan — Fase J: Bestiario D&D 2024

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Bestiario del Monster Manual 2024 consultable en `/bestiario`, con statblocks completos (datos mecánicos) y descripciones propias en español, enlazado a la calculadora de encuentros. Arte de monstruos llega con la Fase H (Storage).

**Architecture:** Datos estáticos en `data/bestiary/` (chunks por CR, patrón `classdata/`), registro con índice. Fuente: PDF del usuario (`C:\Users\carlo\Desktop\Exandria-Obsidian\Exandria\Books\DnD 5e 2024 Monster Manual Alternate Cover.pdf`, 390 págs, capa de texto OCR con ruido — verificar visualmente páginas dudosas renderizando con `py` + `pypdfium2`). Complemento: Monsters of the Multiverse (inglés, texto limpio). Página `/bestiario` con buscador; visibilidad de jugador por lista de "descubiertos" en `app_config` (sin migración).

**⚠️ Convención de contenido (innegociable):** del PDF se extraen SOLO datos
mecánicos — nombre (EN + traducción ES), tamaño, tipo, alineamiento, CA, PG y
su fórmula, velocidades, características, salvaciones, habilidades,
resistencias/inmunidades, sentidos, idiomas, CR/XP/BC, iniciativa, y la
MECÁNICA de rasgos/acciones/reacciones/legendarias (nombre + dados, alcances,
CDs, usos). El texto descriptivo del manual NO se copia ni se traduce: cada
monstruo lleva un `blurb` de 1-2 frases de redacción PROPIA. Los nombres de
rasgos/acciones y los valores numéricos son hechos de juego.

**Tech Stack:** TypeScript estático, Next.js 16. Extracción: `py` + pypdf (texto) / pypdfium2 (render visual). Español.

## Modelo (Task J1)

```ts
// data/bestiary/types.ts
export type MonsterAbility = { name: string; text: string };  // text = mecánica en ES propio, conciso
export type Monster = {
  slug: string;            // kebab del nombre ES
  name: string;            // ES (traducción propia/convencional)
  nameEn: string;          // nombre original (hecho)
  size: string;            // "Mediano", "Grande"…
  type: string;            // "Dragón", "No muerto"…
  alignment: string;
  ac: number; initiative: number;
  hp: number; hpFormula: string;      // "5d8 + 5"
  speeds: string;                      // "9 m, vuelo 18 m"
  abilities: Record<"fue"|"des"|"con"|"int"|"sab"|"car", number>;
  saves?: string; skills?: string;
  resistances?: string; immunities?: string; vulnerabilities?: string;
  gear?: string; senses: string; languages: string;
  cr: string; xp: number; pb: number; // "1/4", 50, 2
  traits?: MonsterAbility[];
  actions: MonsterAbility[];
  bonusActions?: MonsterAbility[];
  reactions?: MonsterAbility[];
  legendary?: MonsterAbility[];
  blurb: string;           // 1-2 frases PROPIAS
  habitat?: string; treasure?: string; // hechos de la cabecera 2024
  image?: string;          // URL Storage (Fase H, después)
};
```

Chunks: `data/bestiary/cr-0.ts`, `cr-18.ts` (1/8), `cr-14.ts` (1/4), `cr-12.ts` (1/2), `cr-1.ts` … `cr-30.ts` — agrupar CRs contiguos si un archivo queda corto (objetivo ~20-40 monstruos/archivo). Índice `data/bestiary/index.ts`: `MONSTERS: Partial<Record<string, Monster>>` + `getMonster(slug)` + `ALL_MONSTERS` + `searchMonsters(q, { cr?, type? })`.

## Pipeline de extracción (Tasks J2..Jn, por lotes)

1. Script una vez (`scripts/extract-mm-text.py` en scratchpad, no commiteado): volcar texto por página del PDF a `scratchpad/mm/pageNNN.txt`.
2. Por lote (subagente): recibe el rango de páginas + los .txt; para páginas con OCR dudoso, renderiza la página (`pypdfium2` scale 2.5 → PNG) y la LEE visualmente para verificar números.
3. Escribe el chunk `data/bestiary/<rango>.ts` siguiendo `types.ts`; registra en índice.
4. `scripts/check-bestiary.ts` (commiteado, crece con cada lote): slugs únicos; para cada monstruo: 6 características 1-30, `xp` == tabla `CR_XP` de data/encounters para su `cr`, `pb` coherente con CR (2 a CR0-4, 3 a CR5-8, … tabla estándar), hpFormula parsea (regex `\d+d\d+([+-]\d+)?`), ac 5-25, blurb no vacío y ≤ 300 caracteres (fuerza concisión y evita prosa copiada).
5. Orden de lotes (utilidad de mesa primero): L1 CR 0–1/2; L2 CR 1–2; L3 CR 3–5; L4 CR 6–10; L5 CR 11–17; L6 CR 18–30. (~6 lotes, 380 págs de statblocks. El apéndice de animales entra en L1-L2.)
